# Spécifications Projet

Ce dossier contient les spécifications fonctionnelles et techniques du projet.

## Utilisation

1. **Copier** le template : `cp _TEMPLATE.md SPEC-001-nom-feature.md`
2. **Remplir** les sections pertinentes (supprimer les sections inutiles)
3. **Soumettre** en PR pour review

## Convention de nommage

```
SPEC-XXX-slug-court.md
```

Exemples :
- `SPEC-001-authentification.md`
- `SPEC-002-dashboard-analytics.md`
- `SPEC-003-notifications-email.md`

## Statuts

| Statut | Signification |
|--------|--------------|
| `draft` | En cours de rédaction |
| `review` | En attente de validation |
| `approved` | Validé, prêt à implémenter |
| `in-progress` | En cours de développement |
| `done` | Implémenté et vérifié |
